traverse(Stations, Users, chtype, varargin)¶

This method applies the channel properties to the receiver modules. The recieved waveform and meaningful physical parameters are written to the receiver module depending on the channel type selected. Two options exist.

Parameters:
  • Stations (enb.EvolvedNodeB) – Station objects with a transmitter and receiver module.
  • Users (ue.UserEquipment) – UE objects with a transmitter and receiver module
  • chtype (str) – the receptive Rx module is configured using ‘downlink’ or ‘uplink’, if ‘downlink’ Users.Rx modules are used to store the final waveforms and physical parameters.
  • varargin – (‘fieldType’,’field’) can be given to bypass fading and interference, ‘full is default’
Stations needs the following fields:
  • Stations.Users: needed for getting link association between stations and users
  • Stations.NCellID: Identifier to link association
  • Stations.Tx: Transmitter module
  • Stations.Rx: Receiver module
Users needs the following fields:
  1. Users.NCellID: Identifier to link association
  2. Users.ENodeBID: Identifier to link association
  3. Users.Tx: Transmitter module
  4. Users.Rx: Receiver module
class ch.SonohiChannel(Param)¶

This is the base coordinator class for the physical channels.

Currently supported models: [winner, eHATA, ITU1546, B2B]

Warning

‘uplink’ is currently only available in B2B mode.

The constructor requires the following options:

Input Param:Parameter struct containing the following:
Param.channel.modeDL:
 (str) Channel model used in downlink. e.g. ‘winner’
Param.channel.modeUL:
 (str) Channel model used in uplink. e.g. ‘B2B’
Param.channel.region:
 (str) Region of channel. This changes based on the channel model as the mapping and definition is considered different. See each model for the configuration of this, e.g. ch.SONOHImodels
Param.Seed:(int) Base seed for the channel
Param.channel.enableFading:
 (bool) Enable/disable fading
Param.channel.enableInterference:
 (bool) Enable/disable interference

downlink(Stations, Users)¶

Standard downlink logic. Output of this function is written into the ue.ueReceiverModule of the users. If the users are scheduled for transmission The downlink logic follows the follow flow

  1. Find links and eNB - UE pairing.
  2. Setup transmission by copying Stations.Tx.Waveform into Users.Rx.Waveform.
  3. Compute link budget and thus calucated receiver power.
  4. If Channel.enableFading, add fading to waveform.
  5. Add AWGN based on thermal noise and receiver power. Compute SNR.
  6. Add propagation delay.
Parameters:
  • Stations (enb.EvolvedNodeB) – Primarily need Stations.Tx Transmitter module
  • Users (ue.UserEquipment) – Primarily need Users.Tx Receiver module

eHATA¶

class ch.SONOHImodels.eHATA.sonohieHATA(Channel, Chtype)¶

This is using an existing implementation of the Extended HATA model as seen here: https://github.com/usnistgov/eHATA The Extended Hata (eHATA) propagation model code was implemented by employees of the National Institute of Standards and Technology (NIST), Communications Technology Laboratory (CTL).

This model uses frequency, distance and a region determined by a string. The regions are given as:

  • ‘Rural’
  • ‘Urban’
  • ‘Dense Urban’
  • ‘Sea’

ITUR1546¶

class ch.SONOHImodels.ITUR1546.sonohiITU(Channel, Chtype)¶

This is using an existing implementation of the ITU-R 1546 model as seen here: https://www.itu.int/en/ITU-R/study-groups/rsg3/Pages/iono-tropo-spheric.aspx

The model is based on frequency, percentage time (defaults to 50%), distance and a propagation region. Currently only land is considered. The regions supported are:

  • ‘Rural’
  • ‘Suburban’
  • ‘Urban’
  • ‘Dense Urban’
